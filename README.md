# amnezia-warp-manager

by [@bandju001](https://github.com/bandju)

Bash-скрипт для управления Cloudflare WARP внутри Docker-контейнеров AmneziaVPN (AmneziaWG).
Маршрутизирует трафик выбранных VPN-клиентов через WARP-туннель.

## Установка и запуск

```bash
bash <(curl -sL https://github.com/bandju/amnezia-warp-manager/releases/latest/download/amnezia-warp-manager.sh)
```

При первом запуске скрипт автоматически:
- установит `wgcf` на хост
- зарегистрирует аккаунт WARP
- поднимет WARP-интерфейс в контейнере
- установит команду `mywarp` для быстрого доступа

Далее для запуска достаточно:

```bash
mywarp
```

Команда `mywarp` каждый раз скачивает актуальную версию скрипта с GitHub — обновление происходит автоматически.

## Поддерживаемые контейнеры

| Контейнер    | Интерфейс | Команда   |
|--------------|-----------|-----------|
| amnezia-awg2 | awg0      | awg-quick |
| amnezia-awg  | wg0       | wg-quick  |

Скрипт ищет контейнеры строго по этим именам — это стандартные имена, которые создаёт AmneziaVPN. Если оба контейнера запущены — предложит выбрать. Если контейнеров нет — сообщит и выйдет.

## Меню

```
╔══════════════════════════════════════════════════╗
║       amnezia-warp-manager  by @bandju001        ║
╚══════════════════════════════════════════════════╝
  Контейнер: amnezia-awg2
  Подсеть:   10.8.1.0/24
  WARP IP:   104.28.225.96

  1) Добавить клиентов в WARP
  2) Убрать клиентов из WARP
  3) Показать клиентов в WARP
  4) Показать статус
  5) Перезапустить контейнер
  6) Полностью удалить WARP
  0) Выход
```

В подменю всегда есть `0) Отмена` для возврата.

## Что умеет

- **Добавление клиентов** — одного, нескольких через запятую, или всех сразу
- **Удаление клиентов** — убирает из WARP сразу и сохраняет в start.sh
- **Имена клиентов** рядом с IP (из clientsTable контейнера)
- **Метка [WARP]** — при выборе видно, кто уже маршрутизируется через WARP
- **WARP Exit IP** — показывает внешний IP WARP-туннеля прямо в меню
- **Персистентность** — список клиентов в `/opt/warp/clients.list`, переживает рестарт
- **Безопасный патчинг start.sh** — блок между маркерами, оригинальная логика не затрагивается
- **Бэкапы** перед изменениями (таймстамп + final-backup для отката)
- **Архитектура** — автоопределение при скачивании wgcf (amd64, arm64, armv7)

## Как это работает

1. На хосте через `wgcf` регистрируется аккаунт Cloudflare WARP и генерируется WireGuard-профиль
2. Профиль копируется в контейнер, поднимается интерфейс `warp`
3. Через `ip rule` + `iptables NAT MASQUERADE` трафик выбранных клиентов направляется в WARP
4. Список клиентов хранится в `/opt/warp/clients.list` внутри контейнера
5. В `start.sh` вставляется блок между маркерами `# --- WARP-MANAGER BEGIN ---` / `# --- WARP-MANAGER END ---`
6. При рестарте контейнера или сервера правила восстанавливаются автоматически

## Полное удаление

Пункт 6 в меню удаляет всё что установил скрипт:

- Интерфейс warp и каталог `/opt/warp/` из контейнера
- Блок WARP-MANAGER из start.sh (восстанавливает оригинал из бэкапа)
- `wgcf`, `wgcf-account.toml`, `wgcf-profile.conf` с хоста
- Команду `mywarp`

После удаления сервер полностью чист — как будто скрипт не запускался.

## Требования

- Linux (root)
- Docker
- curl, wget, awk, sed, grep, getent
- Работающий контейнер `amnezia-awg` или `amnezia-awg2`
